# Мильчин — типограф

Скил-обёртка вокруг детерминированного Python-скрипта. Применяет ТОЛЬКО правила,
которые формализуются без понимания смысла: типографические знаки, неразрывные
пробелы, юникод-гигиена. Стилистику, грамматику, спорную пунктуацию не трогает —
это не его зона.

Имя — в честь Аркадия Эммануиловича Мильчина, автора «Справочника издателя и
автора», канонического свода по русской типографике и оформлению.

---

## Принцип (читать первым)

**Детерминированно — значит без сомнений.** Каждое правило либо однозначно
применимо, либо вообще не применяется. Скрипт никогда «не знает наверняка» —
если правило неоднозначное (десятичный разделитель, минус vs дефис в формуле,
ёфикация), оно просто не реализовано. Лучше пропустить, чем сломать.

**Что под капотом — Python, не LLM.** Это значит:
- 0 токенов
- одинаковый вход даёт одинаковый выход (полная воспроизводимость)
- скорость — миллисекунды на лонгрид
- никаких галлюцинаций

**Защита зон.** До любой правки скрипт прячет в плейсхолдеры:
- YAML-фронтматтер (`---…---` в начале файла — там кавычки/двоеточия это синтаксис)
- inline-code `` `…` ``, fenced-блоки ``` ``` `…` ``` ```
- URL (http/https)
- markdown-ссылки `[текст](url)` — текст типографим, url не трогаем
- @mentions и #hashtags

Это значит, что код, ссылки, метаданные и упоминания **не пострадают** от автозамены
кавычек или тире.

**Markdown-safe.** Маркеры списков (`- пункт` в начале строки) и тире диалогов не
превращаются в em-dash; ведущий отступ строки (вложенные списки) не схлопывается.
В em-dash идёт только дефис между словами в середине строки.

## Триггеры и режимы

| Триггер | Режим |
|---------|-------|
| «прогони через мильчина» \| «типограф» \| «типографика» \| «ёлочки» \| «неразрывные пробелы» \| «/milchin» | **Fix** (default) — исправить и отдать текст |
| «что бы поправил» \| «только проверка» \| «список правок» | **Check** — посчитать, что поправил бы, текст не менять |
| «покажи отчёт» \| «исправь и расскажи» | **Report** — исправленный текст + сводка по правилам |

## Как запустить

Скрипт лежит рядом — `milchin.py` в этой же папке.

**По файлу:**
```bash
python3 milchin.py --file путь/к/файлу.md --fix > путь/к/файлу.md.tmp
mv путь/к/файлу.md.tmp путь/к/файлу.md
```

**По тексту из stdin:**
```bash
echo "Это - тест 5 кг." | python3 milchin.py --fix
# → Это — тест 5 кг.
```

**Только посчитать правки (без изменений):**
```bash
python3 milchin.py --file file.md --check
# → milchin: правок всего 21
# →   T5     15
# →   T9     6
```

**Исправить и показать отчёт:**
```bash
python3 milchin.py --file file.md --report
```

**Прогнать встроенные тесты:**
```bash
python3 milchin.py --selftest
# → 49 PASS, 0 FAIL из 49
```

## Что делает (ряды правил)

### W — юникод-гигиена

- **W9** — NFC-нормализация входа
- **W3** — strip ведущего BOM (U+FEFF)
- **W2** — удалить zero-width / invisible (ZWSP, ZWNJ, WJ, LRM/RLM, math invisibles)
- **W14** — soft hyphen U+00AD убрать
- **W15** — узкие пробелы (U+202F, U+2009, U+2007) → NBSP/обычный
- **W6** — mojibake closed-list (`â€"` → em-dash, `â€™` → апостроф, `Â`+пробел → NBSP)
- **W1** — омоглифы: латинские буквы внутри кириллического токена → кириллица (TR39)
- **W4** — английские «умные» кавычки в русском контексте → ёлочки

### T — базовая типографика

- **T14** — двойные/множественные пробелы → один
- **T15** — пробел перед `.,;:!?»)` → убрать
- **T16** — пробел после `«(` → убрать
- **T24** — пробел после `,;:!?`, если знак слип со словом (Короче,рассказываю → Короче, рассказываю); точку не трогаем (аббревиатуры/расширения)
- **T1** — прямые `"..."` → «ёлочки» (с учётом вложенности → „лапки")
- **T3** — дефис между словами (с пробелами) → em-dash (—); `--`/`---` тоже → em-dash
- **T4** — числовой диапазон `\d-\d` → en-dash (3–5, 1941–1945)
- **D8** — NBSP перед «гг.» после года (1990–1995 гг.)
- **T5** — NBSP после однобуквенных предлогов/союзов (в, к, с, и, а…)
- **T6** — NBSP между числом и единицей/знаком (5 кг, 10 %, № 5)
- **T9** — NBSP перед em-dash
- **T11** — сокращения «т. е.», «и т. д.», «и т. п.», «в т. ч.» → NBSP внутри
- **T12** — многоточие `...` → `…`

### S — научно-техническое (безопасные)

- **S1** — знак умножения единиц `x/х/*` → `·` (только между обозначениями единиц)
- **S6** — угловые градусы/минуты/секунды слитно с числом (20°), но °C/°F не трогаем
- **S23** — № + NBSP перед числом

### Спорные (под флагами)

- `--percent-space` (T7): NBSP перед `%` (по умолчанию OFF — слитно «10%»)
- `--no-initials-space` (T10): не привязывать инициалы NBSP (по умолчанию ON — «А. С. Пушкин»)

## Что НЕ делает

Скрипт сознательно не трогает зоны, где автозамена рискует сломать смысл.
Это работа человека или LLM-скила (`/rozental` для нормы, `/chukovsky` для стиля):

- **Минус U+2212 vs дефис в формулах/температуре** — неотличимы без контекста
- **Разряды больших чисел** — риск задеть годы, телефоны, ID
- **Десятичный разделитель `.` → `,`** — неотличим от версий/дат/IP
- **Прямая речь и перестановка знаков у кавычек** — требует анализа
- **Буква ё** — редполитика, не норма
- **Наращения порядковых** (1-й/2-го) — стилистика
- **Орфография, пунктуация по смыслу, согласование, управление** — для этого `/rozental`
- **HTML-сущности** — зависит от parse_mode публикации
- **Эмодзи VS-нормализация** — стиль

## Формат вывода

```
milchin: правок всего {N}
  T5     15
  T9     6
  W2     2
```

В режимах `--fix` и `--report` исправленный текст идёт в stdout, сводка — в stderr.
В режиме `--check` stdout пуст, только сводка в stderr.

---

## Калибровка

- **Бесшумно по умолчанию.** Если правок нет — печатает только `milchin: правок нет` в stderr.
- **Чистый stdin/stdout pipeline.** Подходит для unix-композиции: `cat | milchin --fix | dest`.
- **Безопасно на чистом тексте.** На уже типографированном тексте правок ноль — скрипт идемпотентен.
- **49 встроенных тестов** (`--selftest`). При любых правках в скрипте — прогонять.

## Смежное

- Норма (орфография, пунктуация, согласование) → `/rozental`
- Смысл, структура, голос → `/chukovsky`
- AI-маркеры и нейрослоп → `/slopotron`

Канонический порядок: **Чуковский → Розенталь → Слопотрон → Мильчин**
(смысл → буква → детектор → форма).
