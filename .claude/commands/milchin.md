Используй скил `milchin`. Это не LLM-задача — это запуск Python-скрипта
`milchin.py` (лежит в `.claude/skills/milchin/`).

**Если в запросе указан файл** — прогони:

```bash
python3 .claude/skills/milchin/milchin.py --file <путь> --report
```

(Если файла нет в этом скоупе, поищи `~/.claude/skills/milchin/milchin.py`.)

В `--report` исправленный текст идёт в stdout, сводка по правилам — в stderr.
Покажи пользователю сводку (`milchin: правок всего N + по рядам T/D/W/S`),
спроси «применить к файлу?» — и только после «да» перезапиши.

**Если в запросе только текст** — пайплингом:

```bash
cat <<'EOF' | python3 .claude/skills/milchin/milchin.py --report
<текст>
EOF
```

Покажи исправленный текст + сводку.

**Если в запросе явно сказано** «только проверка» / «что бы поправил» / «список» —
используй `--check` (текст не меняется, только счётчики в stderr).

Главное правило: скрипт **не понимает смысла** — он применяет только однозначные
правила типографики и юникода. Если пользователь спрашивает про орфографию,
пунктуацию-по-смыслу, грамматику — это не Мильчин, это `/rozental` (или другой
LLM-скил).
